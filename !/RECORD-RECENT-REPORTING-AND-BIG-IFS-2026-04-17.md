# Record - Recent Reporting and Big IFs

## Scope

This record consolidates the recent reporting into one filed surface.

It does not supersede the individual notes it cites.
It reduces the current run to one operational spine, one targeted research
slice, and one set of durable `Big IFs`.

## Recent Reporting Spine

The recent reporting now resolves into one coherent sequence.

### 1. The studio packet established the genre posture

The four-note packet established a stable read:

- civic fantasy rather than generic fantasy
- standing rather than generic trust
- lawful return rather than endless depth-seeking
- syntax, count, and placement as operational law rather than ornament

Those notes were strong enough to stop being only workshop talk.
They now function as a real design packet.

### 2. The dev-design report named the truthful implementation seam

The follow-on development report narrowed the first executable seam to a
bare-bones scaffold grounded in existing Python machinery.

That report did two useful things:

- it refused to overclaim that the full standing engine already exists
- it identified the current local machinery that can truthfully carry a first
  scaffold

The key move there was restraint:
wrap the live machinery first, do not narrate a larger runtime than the repo
can presently prove.

### 3. The tool-lane family matured from essay to instrument

The tool-lane draft family now has a clean developmental read:

- first draft = problem charter
- first map = worked pilot
- v2 map = reusable template

That family matters because it turned framework sprawl into a routing problem
with closed vocabularies, step-level driver lanes, and support lanes that no
longer pretend to be co-equal.

### 4. The cabinet signal corrected the read on `CROSSFRAMING-US`

The later signal matters because it corrected the task at the right level.

`CROSSFRAMING-US` is not primarily a note-family comparison problem.
It is a Python framing problem.

The House does not mainly need more prose about its framing.
It needs a small typed layer that can hold:

- closed vocabularies
- strict models
- validators
- renderers
- reusable loop presets

That is the cleanest current bridge between workshop notes and executable
discipline.

## Targeted Research

### Necromancy and Necromancers

The local corpus gives a more useful read on necromancy than a generic
"death magic" label would.

Across `- Fandom, Inc. - Necromancer.md`, `- Fandom, Inc. - Free Magic.md`,
`- Fandom, Inc. - The Passage.md`, and `- Fandom, Inc. - Abhorsen.md`, the
pattern is consistent:

- necromancy is linked to unconstrained or severed power rather than ordered
  binding
- necromancers fear lawful death and seek continuity beyond their proper term
- persistence is achieved through distortion, anchoring, replacement bodies,
  or catastrophic scale plans
- the counter-office is not generic heroism but lawful boundary-keeping

The strongest evidence from the local notes is structural, not aesthetic.

`- Fandom, Inc. - Necromancer.md` presents the necromancer as a feared
Free-Magic sorcerer whose connection to the Charter is severed and whose fear
of death drives increasingly illicit persistence.

`- Fandom, Inc. - Free Magic.md` reinforces that necromancy belongs to an
unconstrained magic that corrodes bodies, burns language, and ultimately
devours those who try to master it.

`- Fandom, Inc. - The Passage.md` shows the higher-order necromantic fantasy:
not merely raising the dead, but trying to dam the stream between life and
death itself through mass death in the name of false peace.

`- Fandom, Inc. - Abhorsen.md` gives the lawful counter-form:
an office that combines force and order to maintain the border, correct
perversion, and keep the line between life and death intact.

That means the local necromancy material is directly useful to the project's
standing-and-endings model.

The real design inheritance is:

- necromancy = persistence beyond sanction
- lawful office = witnessed maintenance of the boundary
- lich-work = continuity mistaken for legitimacy

This is a strong fit for the House because it supports the anti-lich logic
already emerging in the packet:

survival is not sanction
persistence is not standing
memory is not authority

## Big IFs

### BIG IF 1: The packet is now strong enough to govern implementation tone

The civic-fantasy packet is no longer just a good conversation.
It is strong enough to act as a design constraint on future implementation
choices.

Its value is not that it explains everything.
Its value is that it narrows drift.

### BIG IF 2: The next real seam is Python framing, not more metaphysical prose

The recent reporting converges on one practical conclusion:
the next material gain is a small Python framing layer that can carry route,
standing, lane, and output judgments with more rigor than prose notes alone.

Notes should witness the system.
Python should increasingly carry the constraint grammar.

### BIG IF 3: The tool question is finally in the right form

The useful question is no longer "what frameworks exist?"
It is:

which lane satisfies which use case under which lifecycle posture and
permission boundary.

That is a materially better decision frame than framework shopping.

### BIG IF 4: One driver lane per step is a governance primitive, not just a formatting preference

The move from multiple preferred lanes to one driver lane per step is a real
House rule in embryo.

Without it, agents can always justify the lane they already wanted.
With it, routing becomes reviewable.

### BIG IF 5: Necromancy is already coded in the local corpus as illicit persistence

The necromancy notes matter because they provide a strong local analogy for the
Vault's own lawful-endings logic.

The core problem is not "dark spooky power."
It is action that continues after rightful standing has ended and begins to
treat continuity as authority.

That is the same structural problem the anti-lich design is trying to name.

### BIG IF 6: Maintenance offices are part of world truth, not mere cleanup chores

The recent residue reads point in the same direction:
zombie traces, mojibake, stale structural drift, and haunted draft residue are
not only repository mess.
They are recurring classes of failure in a record-heavy world.

That suggests the House will eventually need explicit maintenance offices or
job duties for:

- burial
- exorcism
- encoding repair
- residue classification

Not because the metaphor is cute, but because the failure modes are real and
repeatable.

## Bottom Line

The recent reporting now says one thing clearly.

The House has enough language.
What it needs next is stricter framing.

The packet gave the world a usable law-shape.
The maps gave route discipline a usable instrument.
The signal corrected the target from essay to Python.
The necromancy read reinforced that lawful endings and illicit persistence are
already native to the local corpus.

That is a coherent basis for the next build seam.
