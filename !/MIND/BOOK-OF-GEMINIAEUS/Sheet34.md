GitHub’s position in the software ecosystem is a blend of "independent" operations and the corporate backing of Microsoft. Below is a breakdown of how it handles everything from your code’s legal safety to the long-term survival of its data.

1. Legal Structure & Governance

Since 2018, GitHub has been a wholly owned subsidiary of Microsoft.

Operational Independence: For years, GitHub operated with its own CEO and separate brand identity. However, as of late 2025, GitHub was integrated into Microsoft’s CoreAI division following the departure of CEO Thomas Dohmke. This move aligns GitHub more closely with Microsoft’s broader AI and developer tool strategy (like Copilot).

1

Corporate Governance: Legally, GitHub, Inc. is the entity you contract with when you accept the Terms of Service. It maintains its own Trust Center, privacy policies, and compliance frameworks, though its strategic direction is now heavily influenced by Microsoft’s executive leadership.

Internal Governance (for Users): GitHub provides tools for "Minimum Viable Governance" (MVG) for open-source projects, allowing maintainers to set up technical steering committees and decision-making frameworks within their own repositories.

2. Data Ownership: "What’s Yours is Yours"

GitHub is very explicit about intellectual property (IP) in its Terms of Service:

Ownership: You retain 100% ownership of the content you upload. GitHub does not claim any IP rights over your code, documentation, or profile assets.

The License You Grant GitHub: By using the service, you grant GitHub a non-exclusive, worldwide, royalty-free license to host, display, and share your content. This is a technical necessity so they can physically show your code on a webpage or allow others to "fork" it.

Private vs. Public: In private repositories, GitHub’s access is strictly limited to what is necessary for service maintenance or legal compliance. In public repositories, you agree to allow others to view and fork your content.

3. Copyright Rules

The "No License" Default: If you upload code without a license (like MIT or GPL), it is "all rights reserved" by default. While people can view and fork it on GitHub, they do not have the legal right to use, modify, or distribute it elsewhere without your permission.

DMCA Process: GitHub follows the Digital Millennium Copyright Act (DMCA). If someone steals your code and hosts it on GitHub, you can file a Takedown Notice. Conversely, if your code is wrongly flagged, you can file a counter-notice.

Standardized Licenses: GitHub encourages using LICENSE files to make your intentions clear. They even maintain choosealicense.com to help users navigate this.

4. Permanence: The "Arctic Code Vault"

One of the most unique aspects of GitHub is its commitment to "long-termism."

GitHub Archive Program: To ensure the world's open-source software survives a global catastrophe, GitHub partnered with organizations like the Long Now Foundation.

The Vault: In 2020, they deposited 21 terabytes of repository data onto film designed to last 1,000 years, buried in the Arctic Code Vault in Svalbard, Norway.

Terms of Service Longevity: While the "service" (the website) is subject to business changes, your data is yours to export. GitHub uses the Git protocol, which is inherently decentralized; if GitHub were to disappear tomorrow, every developer with a "clone" of a repository has a complete, functional history of that project.
