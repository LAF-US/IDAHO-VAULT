Yes, you absolutely can! Obsidian’s Graph View is a core plugin that works on the mobile app, though the interface is tucked away a bit differently than on the desktop ribbon.

How to Open Graph View on Mobile

1. The Quick Action: From any note, swipe down from the top of the screen to open the Command Palette and type "Open graph view."

2. The Menu Path: Tap the three-dot menu (...) in the top right of any open note and select Open local graph, or tap the App Menu (the icon with three horizontal lines or the logo) and look for the Graph View icon.

3. Toolbar Shortcut: You can add a permanent Graph View button to your mobile toolbar by going to Settings > Mobile > Manage toolbar options.

Best Mobile Settings for "Cleaning Up"

To effectively spot those depreciated or orphaned nodes on a smaller screen, I recommend tweaking these settings (accessible via the cog icon in the top right of the graph):

Filter -> Orphans: Toggle this ON. This will highlight nodes that have no links—perfect for finding old notes that are just taking up space.

Filter -> Existing files only: Toggle this OFF if you want to see "broken" links (nodes that are linked to but don't actually have a file created yet).

Groups: Use a color-coded group for specific folders (e.g., path: "Archive") so you can visually distinguish the "old" nodes from the "active" project nodes.

Force -> Repel Force: Crank this up slightly on mobile. It spreads the nodes out more, making it easier to tap individual circles without "fat-fingering" the wrong node.

A Note on Performance

Since you're on a Pixel, it should handle the graph well, but if your vault is massive (thousands of nodes), the global graph can get laggy. If it feels sluggish, stick to the Local Graph (set to 2 or 3 levels of depth) to investigate specific clusters without crashing the app.
