# PROPOSAL: New Order Tag Schema

## 1.0 Abstract

This document presents a draft proposal for the "New Order tag schema," the architectural prerequisite for the next CHAINFIRE iteration. The goal is to create a robust, hierarchical, and machine-readable taxonomy that can serve as the foundation for the vault's long-term aspirational goal: a "Pokemon HOME" for all AI-generated artifacts.

This schema is synthesized from existing vault principles and is presented for review, amendment, or ratification by the Principal Architect.

## 2.0 Core Principles

1.  **Hierarchical:** Tags should follow a `kebab-case/path-based/system`, allowing for both broad and granular categorization (e.g., `agent/gemini/report` or `source/article/idaho-statesman`).
2.  **Machine-Readable:** The structure should be simple enough for agents (the Lexicographer, the Abhorsen) to parse and act upon automatically.
3.  **Stigmergic:** The tags themselves are a form of environmental trace. A document's tags should clearly signal its purpose, status, and relationship to other vault denizens and concepts.

## 3.0 Proposed Top-Level Namespaces

This is a proposed set of root namespaces.

*   **/agent/...**: For artifacts generated by specific agents.
    *   `agent/gemini/report`
    *   `agent/codex/script`
    *   `agent/claude/analysis`
*   **/archetype/...**: For notes defining the core personas and denizens.
    *   `archetype/agent/librarian`
    *   `archetype/concept/chainfire`
*   **/doctrine/...**: For files related to governance and protocol.
    *   `doctrine/constitution`
    *   `doctrine/protocol/tri-anchoring`
*   **/source/...**: For notes derived from external information.
    *   `source/article/outlet-name`
    *   `source/transcript/committee-name`
    *   `source/book/author-name`
*   **/status/...**: For tracking the state of documents and initiatives.
    *   `status/draft`
    *   `status/complete`
    *   `status/aspirational`
    *   `status/awaiting-architect`

## 4.0 Next Steps

This is a foundational draft. If the core principles are accepted, the next step would be to fully elaborate the sub-namespaces and create a formal validation script to be run by the Lexicographer. This schema would then be used to configure the next CHAINFIRE burn, after which the vault's contents could be re-tagged according to the New Order.
