No degradation in storage.\n\n  \n\n**Organizational features:**\n\n- Boxes can be named and assigned wallpapers/themes.\n\n- Modern games add sorting, searching, and filtering across boxes.\n\n- The distinction between \"in party\" and \"in storage\" is hard-architectural: the game engine treats party and PC as separate data structures.\n\n  \n\n**Evolution of the mechanic:**\n\n| Era | System | Access Model |\n\n|---|---|---|\n\n| Gen 1–5 (1996–2012) | Local PC (per cartridge) | Fixed terminal; data stays on cartridge |\n\n| Gen 6–7 (2013–2019) | Pokemon Bank | Cloud service; cross-game deposit/withdraw |\n\n| Gen 8–present (2020–) | Pokemon HOME | Unified cloud registry; cross-game, cross-platform, persistent entity layer |\n\n  \n\n**Pokemon HOME** represents the mature form: a durable entity registry independent of any single game session. Pokemon from Gen 1 (via Virtual Console) can be migrated forward. The entity persists across games, consoles, and years.\n\n  \n\n---\n\n  \n\n### Minecraft Ender Chest\n\n  \n\n**Game:** Minecraft Java Edition 1.3.1 (2012).
