# FANDOM Canon Companion Coverage

The canonical human-readable surface for all six canon sets is
`FANDOM-CANON-RULESYSTEMS-2026-05-03.md`. The JSON file is the machine source
validated by `.github/scripts/check_fandom_canon_rulesystems.py`.

Separate companion notes are optional interpretive surfaces. They should be
added when a canon needs vault-facing doctrine beyond the registry summary, not
as mandatory boilerplate.

| Canon | Registry rulesystem | Companion coverage |
| --- | --- | --- |
| Bartimaeus | `binding-ontology` | `TRUE-NAME-EXPLORER-COMPANION-2026-04-14.md` and related plane/name notes cover identity and binding details. |
| Snicket / ASOUE | `family-institution-index` | Denouement, Sugar Bowl, Underwater Library, and related explorer companions cover archival/institutional surfaces. |
| Children of the Lamp | `tribe-vessel-law` | `CHILDREN-OF-THE-LAMP-EXPLORER-COMPANION-2026-05-03.md` plus vessel/tribe analysis notes. |
| Old Kingdom | `charter-free-magic-law` | Registry summary is currently sufficient; add a companion only if Charter/Free Magic enforcement gets expanded. |
| Skulduggery Pleasant | `hidden-society-discipline-law` | `LANDY-STROUD-CANON-MATRIX-RECONCILIATION-2026-05-03.md` covers the Landy/Stroud bridge; registry summary covers the base canon. |
| Avatar | `spirit-cycle-continuity-law` | `AVATAR-CANON-RULESYSTEM-COMPANION-2026-05-04.md` covers succession, incarnation, memory bridge, and severance risk. |

## Boundary

FANDOM canon companions explain literary rulesystems. They do not define persona
chamber standards, Touchstone Tree noun standards, or connector posture.

When those systems share an enforcement pattern, preserve the distinction:

- FANDOM registry: canon rulesystems and source-backed primitives
- Persona standards: Vault Agent continuity, chambers, and runtime/tooling splits
- Tree noun registry: Touchstone noun anchors and `NOUN.md` / `.noun/NOUN.md`
  pairs
