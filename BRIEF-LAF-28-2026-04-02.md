# BRIEF - LAF-28 Control-Plane Triptych

## Summary

LAF-28 is the scoped control-plane issue for local bootstrap and cross-system operating clarity.

It does **not** reopen the full multi-plugin orchestration question. This issue hardens the local bootstrap path and states the current division of labor across the active systems.

## Authority Chain

For this lane:

1. `!/AGENTS.md` is the canonical narrative registry.
2. `swarm.json` is the canonical machine-readable registry.
3. `!/agents.json` is the derived bootstrap index for `!/agent.sh`.
4. `!/agent.sh` is the executable local bootstrap entrypoint.
5. `LAF-25` remains the coordination hub.
6. `LAF-28` is the scoped implementation issue for this triptych.

## The Triptych

### GitHub

GitHub handles:

- branches
- pull requests
- workflows
- durable execution transport

GitHub is an execution surface, not vault doctrine.

### Linear

Linear handles:

- issue state
- routing
- handoffs
- active coordination

`LAF-25` is the hub. `LAF-28` is the scoped control-plane child lane.

### Slack

Slack remains:

- breadcrumb-only
- ephemeral
- non-authoritative

Durable decisions must still be promoted into vault files or Linear.

### Hugging Face

Hugging Face remains available for:

- model lookup
- dataset lookup
- documentation
- jobs

For this issue, it is an authenticated **read-only sidecar**. It is not part of local bootstrap and not part of live coordination writes.

## Bootstrap Implications

`!/agent.sh` now resolves agents through generated `!/agents.json` instead of hardcoded shell tables.

The required local context bundle is:

- `!/README.md`
- `!/CONSTITUTION.md`
- `!/AGENTS.md`
- `!/DECISIONS.md`
- `!/VAULT-CONVENTIONS.md`

Optional advisory context:

- `!/LEVELSET.md`

`!/LEVELSET-CURRENT.md` is not part of the active bootstrap path.

## Boundaries

This issue does not:

- add Linear auth to `!/agent.sh`
- add Slack auth to `!/agent.sh`
- add Hugging Face auth to `!/agent.sh`
- promote Hugging Face into a first-class control-plane adapter
- replace the vault as source of truth

## Definition Of Done

LAF-28 is satisfied when:

1. `swarm.json` carries the control-plane block and bootstrap metadata.
2. `!/agents.json` is generated from `swarm.json` and clearly marked derived.
3. `!/agent.sh` reads the generated index and supports read-only inspection.
4. `!/AGENT-PROTOCOL.md` matches the real implementation.
5. A new engineer can inspect the bootstrap path and system roles without guessing.
