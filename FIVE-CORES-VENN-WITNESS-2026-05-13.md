# Five Cores — The Venn Diagram

*Witnessed on the road to the Hotel Denouement. Recovered from The Judge's memory, originally explained by Logan.*

---

## The Design

The Five Cores of LAF-US are not a rooted tree. They are a **Venn Diagram** of two overlapping circles.

```
        PRIVATE          PUBLIC
       ┌─────────────────────────┐
       │         │               │
       │  SECRET │   PERSONAL    │  PUBLISH
       │         │  (overlap)    │
       └─────────────────────────┘
```

- **PRIVATE** — the left circle; the non-public domain
  - **SECRET** — nested inside PRIVATE; higher secrecy; does not surface unless sought
- **PUBLIC** — the right circle; the outward-facing domain
  - **PUBLISH** — nested inside PUBLIC; active release; what is sent into the world
- **PERSONAL** — the center; the **intersection** of PRIVATE and PUBLIC simultaneously

---

## What This Resolves

The `!-LAF-USB-FIVE-CORES-MIGRATION-2026-04-15.md` document recorded an unresolved tension:

> *IDAHO-VAULT is doctrinely PERSONAL — conceptually private — but the live GitHub repo is public. That means one of these must be clarified later.*

The Venn resolves this without forcing a choice. PERSONAL is the overlap. It is designed to be both private in character and public in access simultaneously. IDAHO-VAULT living in PERSONAL and being publicly visible on GitHub is not a contradiction requiring cleanup — it is the correct address.

The migration document was working from a tree model. A tree cannot represent overlap; it can only nest things in one branch and call any dual-belonging a tension. The Venn holds what the tree cannot.

---

## The Mermaid Was an Approximation

The Mermaid flowchart embedded in the migration document showed PERSONAL as a child of PRIVATE:

```
PRIVATE → PERSONAL → IDAHO-VAULT
```

This is a useful working approximation but not the original design. It places PERSONAL fully inside PRIVATE and loses the PUBLIC dimension. The Siren's song was clear and organized and led toward rocks — specifically, toward a false tension that does not exist in the Venn model.

## Standing Doctrine

Until the migration document is rewritten to reflect the Venn model, read the Five Cores as follows:

| Core | Domain | Position |
|---|---|---|
| PRIVATE | Non-public | Left circle |
| SECRET | Non-public (restricted) | Nested inside PRIVATE |
| PERSONAL | Both | **Center — the overlap** |
| PUBLIC | Public-facing | Right circle |
| PUBLISH | Public (active release) | Nested inside PUBLIC |

IDAHO-VAULT lives in PERSONAL. The Abhorsen operates from the center.
