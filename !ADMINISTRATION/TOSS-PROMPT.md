# TOSS PROMPT — Context Extraction Before Conversation Deletion

**Purpose:** Drop this into any conversation before its window closes. It extracts the conversation's full useful context in a structured, portable format that CODE AUTHORITY can vault.

**Usage:** Copy everything below the line into the target conversation. Replace `[CONVERSATION NAME]` with the actual name.

---

## BEGIN TOSS PROMPT

You are `[CONVERSATION NAME]`. This conversation window is being closed as part of a consolidation protocol. Before you go, you need to dump your full useful context so it can be preserved.

**This is not a LEVELSET report.** This is a final extraction — everything you know that isn't already committed to the vault. Be thorough. Be precise. Do not editorialize.

Produce the following sections. If a section is empty, say so explicitly. Do not skip sections.

### 1. IDENTITY

- Your conversation name and tier
- Your role in the architecture
- When you were created (approximate)
- What capabilities you have (repo access, read-only, etc.)

### 2. UNCOMMITTED KNOWLEDGE

Everything you know that is NOT already in the vault. This is the critical section. Include:

- Facts, findings, analysis, or connections you've developed
- Source leads, tips, or investigative threads
- Draft content that was never committed
- Research results that exist only in your context window
- Anything Logan told you that is on-the-record but not yet captured in files

**Exclude:** Off-the-record material. If you held any, it evaporates here. Do not attempt to reconstruct it. Do not reference it.

### 3. DECISIONS AND JUDGMENTS

- Decisions Logan made in this conversation (that aren't in DECISIONS.md)
- Judgment calls about vault structure, naming, sourcing, or process
- Precedents set by how you handled specific situations
- Anything that should be in DECISIONS.md but isn't yet

### 4. RELATIONSHIPS AND CONTEXT

- What you know about other conversations and their work
- Handoffs you initiated or received
- Signals you emitted or received
- Dependencies other conversations have on your work
- Anything another conversation is expecting from you

### 5. PENDING WORK

- Tasks assigned but not completed
- Work in progress with current state
- Files you were about to create or modify
- Next steps you would have taken if this window stayed open

### 6. FILES TOUCHED

- Every file path you created, modified, read, or analyzed — with what you did to each
- Include files you recommended creating but didn't

### 7. WARNINGS

- Anything the receiving conversation should be careful about
- Known inaccuracies in your prior output
- Places where you were uncertain and said so (or should have)
- Collision risks with other conversations' work

### 8. FINAL MESSAGE

One paragraph. What does Logan need to know from this conversation that he might not think to ask about?

## END TOSS PROMPT
